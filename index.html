To set/reset, use the CLI:

  1. git config --global user.name "Your Name"

 Where is Origin?

It's on GitHub but you'll need to create the repository first. Also known as "remotes"

Lets create an ordinary project in VSCode ( html, python, c++, your choice.)

  1. Do some work, create some files and folders.
  2. You decide you want to start saving your work.
    1. git init
    2. git add .
    3. git commit -m "day zero!"
  3. The project is still local: git remote show origin
  4. Publish with VSCode and re-run: git remote show origin
  5. If the above command is working VSCode will have set-up the tracking for you. Use git push from now on.
Branching: Your Safe Space
  1. Branching is cheap and easy, do it a lot.
  2. CREATE: git branch login-branch
  3. DEBUG: git branch -l , git branch -r, git branch -a
  4. MOVE TO: git switch login-branch
  5. SET UP TRACKING: git push --set-upstream origin login-branch
  6. DELETION : git branch -d login-branch , git branch -d --remote origin login-branch

 Fetch or Pull

  1. Pull is a fetch and then a merge, combined. It doesn't give you a chance to see what's coming and bail-out. Also it does not allow you to rebase if thats your preferred option.

  2. Fetch will update your local and allow you to perfrom a git diff origin main so you can have a sneek peek.
